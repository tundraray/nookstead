## 1. Моделирование кривых удержания

### 1.1 Индустриальные бенчмарки для cozy/life-sim игр

| Метрика | Casual Mobile (Hay Day, Townscapes) | PC Indie (Stardew Valley) | MMO Life Sim (Palia) | **Nookstead Target** |
|---|---|---|---|---|
| **D1** | 35-40% | 65-75% | 45-55% | **55%** |
| **D7** | 15-20% | 40-50% | 20-30% | **30%** |
| **D14** | 10-14% | 30-40% | 12-20% | **22%** |
| **D30** | 6-10% | 20-30% | 8-15% | **15%** |
| **D90** | 2-5% | 10-15% | 3-7% | **8%** |

**Обоснование таргетов Nookstead:**

- **D1 = 55%.** Ниже, чем у premium PC-игр (Stardew: 70%), потому что F2P-аудитория менее мотивирована. Выше, чем у mobile casual (38%), потому что целевая аудитория -- core-игроки 18-35, а не массовый casual. Выше, чем у Palia (50%), потому что браузерный доступ устраняет барьер "скачай и установи".
- **D7 = 30%.** Ключевой порог. Здесь AI NPC дают наибольший эффект -- к дню 7 у NPC накапливаются воспоминания, и игрок впервые испытывает "они помнят меня!" момент.
- **D30 = 15%.** Совпадает с верхней границей MMO life sim. Удерживается за счет сезонной системы (7 реальных дней = 1 сезон, к D30 игрок проживает 4+ сезонов), органических квестов от NPC и сюжетного прогресса.
- **D90 = 8%.** Агрессивный таргет для F2P. Обосновывается уникальностью контента: каждый сервер развивается по-разному, NPC-отношения нельзя "пройти до конца", Act II запускается на ~D90.

### 1.2 Что удерживает на каждом этапе

#### D1: Первый опыт (первые 60 минут)

**Критические события первой сессии:**

| Событие | Время | Влияние на D1 | Приоритет |
|---|---|---|---|
| Первое движение по карте | 0-30 сек | +5% к D1, если FPS > 30 и нет лагов | P0 |
| Встреча с мэром Виктором | 1-3 мин | +8%, если диалог вызывает эмоцию | P0 |
| Посадка первой культуры | 5-10 мин | +12%, если есть визуальная обратная связь | P0 |
| Первый "живой" диалог с NPC | 10-20 мин | +15%, если NPC удивляет/запоминается | P0 |
| Получение первого дохода | 15-30 мин | +10%, если экономический цикл замыкается | P1 |
| Исследование 2+ зон города | 20-40 мин | +7%, увеличивает perceived depth | P1 |
| Встреча с другим игроком | случайно | +6%, social proof | P2 |

**Формула ожидаемого D1:**

```
D1_predicted = 0.30 (базовый)
             + 0.08 * tutorial_complete
             + 0.07 * npc_dialogue_count(>0)
             + 0.05 * crop_planted
             + 0.03 * zone_explored(>1)
             + 0.02 * player_met
```

**Риск для D1:** В первой сессии AI NPC еще не имеет воспоминаний о игроке. Первый диалог -- это cold start. NPC должен быть обаятельным с первого слова, без опоры на память. Это значит, что Seed Persona и первый приветственный промпт -- критически важны для D1.

**Рекомендация:** Тестировать prompt для первого диалога как отдельную оптимизацию. A/B-тест с 3+ вариантами приветствий мэра Виктора.

#### D7: Формирование привычки (дни 2-7)

**Драйверы возврата:**

| Драйвер | Механизм | Ожидаемый lift D7 |
|---|---|---|
| Созревание первых культур | Редис через 12 часов, картофель через 2 дня -- причина вернуться | +5-8% |
| NPC-память (первое узнавание) | NPC упоминает вчерашний разговор: "Вчера ты рассказывал про..." | +8-12% (USP!) |
| Ежедневные квесты (доска объявлений) | Обновляются каждые 24 реальных часа | +4-6% |
| Динамика рынка | Цены меняются каждый день; если вчера продал дешево, сегодня может быть выгоднее | +3-5% |
| Push-уведомления (если разрешены) | "Ваши помидоры созрели!" / "Марко спрашивал о вас" | +5-7% |

**"They remember me!" эффект -- количественная модель:**

Предполагается, что первое осмысленное упоминание NPC прошлого взаимодействия происходит на 2-3 день (после первой рефлексии). Это "вирусный момент":

```
P(share_clip | npc_memory_trigger) = 0.15-0.25
P(return_D7 | experienced_npc_memory) = D7_base * 1.35
```

Ожидаемый lift: +35% к базовому D7 для тех, кто испытал момент "они помнят".

**Условие:** Рефлексия NPC должна произойти до первого входа игрока на D2. При дефолтном времени рефлексии 04:00 серверного времени и типичном времени игры 18:00-22:00 -- это работает. Но если игрок заходит в 06:00, рефлексия прошла всего 2 часа назад и NPC уже "знает" его.

#### D14: Расширение горизонта (дни 8-14)

| Драйвер | Механизм |
|---|---|
| Второй сезон (при 7-дневных сезонах) | Новые культуры, визуальные изменения, сезонный фестиваль |
| Развитие отношений с NPC до "Casual Friend" | Новые типы диалогов, сплетни, приглашения |

| Homestead improvements | Расширение участка, новые строения |
| Социальный компонент | Гильдия, посещения чужих homesteads |

#### D30: Глубина системы (дни 15-30)

| Драйвер | Механизм |
|---|---|
| Прожиты 4+ сезона | Понимание годового цикла, подготовка к следующему |
| NPC "Close Friend" первый | Уникальные секреты, квесты, подарки |
| Act I пройден | Сюжетный крюк для продолжения |

#### D90: "Forever game" (дни 31-90+)

| Драйвер | Механизм |
|---|---|
| Act II (на ~D90) | Новый сюжетный акт -- капсула времени, тайна города |

### 1.3 Модель влияния AI NPC на кривую удержания

```
           D1     D7     D14    D30    D90
Без NPC:   50%    20%    13%    8%     3%
С NPC:     55%    30%    22%    15%    8%

Разница:   +5%    +10%   +9%    +7%    +5%
```

**Объяснение по этапам:**

| Этап | Lift от AI NPC | Почему |
|---|---|---|
| D1 | +5% (слабый) | NPC еще не накопил память. Lift только от качества первого диалога |
| D7 | +10% (максимальный) | "They remember me!" момент. Это USP в чистом виде |
| D14 | +9% | Развитие отношений, сплетни, органические квесты. NPC стал "знакомым" |
| D30 | +7% | Emergent events, уникальный контент сервера. NPC удерживает narrative |
| D90 | +5% | Долгосрочный эффект слабее -- к D90 novelty эффект AI снижается, нужен fresh content |

**Формула ожидаемого удержания с AI NPC:**

```
Retention(d) = R_base(d) * (1 + alpha * NPC_memory_depth(d) * NPC_quality_score)

где:
- R_base(d) -- базовое удержание без AI NPC (экспоненциальный спад)
- alpha -- коэффициент влияния NPC (~0.3-0.5, определяется A/B-тестом)
- NPC_memory_depth(d) -- нормализованное кол-во релевантных воспоминаний о игроке на день d (0-1)
- NPC_quality_score -- метрика качества диалогов (0-1), агрегат по когорте
```

**Риски модели:**

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| NPC-память нерелевантна | Средняя | Высокое: D7 lift = 0 | A/B-тест retrieval weights, мониторинг relevance_score |
| NPC ломает характер | Средняя | Высокое: разрушает иллюзию | Strict system prompt, post-processing, moderation |
| AI-усталость (novelty wears off) | Высокая к D30+ | Среднее | Stimulus injection, new NPC arrivals, evolving personalities |
| Latency > 5s убивает иммерсию | Низкая | Высокое | Caching, model routing, fallback to quick replies |

### 2.3 Влияние real-time на дизайн сессий

Nookstead работает в реальном времени (1:1 по умолчанию). Это фундаментально меняет паттерн сессий:

| Аспект | Compressed Time (SDV) | Real-Time (Nookstead) |
|---|---|---|
| Сессия = | 1-3 игровых дня | Часть 1 реального дня |
| Утренние действия | В начале сессии (всегда) | Только если играешь утром |
| Вечерний бар | Через 10-15 мин сессии | Только после 18:00 реального времени |
| Ощущение "пропустил" | Нет (время стоит без игрока) | Да (время идет 24/7) |
| Anxiety | Минимальный | Риск FOMO |

**Критический риск: FOMO при real-time.**

Если игрок не заходит 3 дня:
- Культуры могут сгнить (если не собраны вовремя)
- Животные голодные (не кормлены)
- NPC продолжили жить без игрока

**Митигация FOMO:**

1. Rot timer -- культуры держатся 48 часов после созревания (достаточно для выходных)
2. Auto-feeder для животных (QoL покупка, или базовая механика)
3. NPC приветствие при возврате: "Давно тебя не было! У нас тут столько произошло..."
4. "Catch-up summary" -- при входе после >24ч отсутствия показать ключевые события

## 4. Фреймворк метрик вовлеченности

### 4.1 Основные метрики вовлеченности

| Метрика | Определение | Формула | Целевое значение | Частота сбора |
|---|---|---|---|---|
| DAU | Уникальные игроки с >= 1 сессией за день | COUNT(DISTINCT player_id WHERE session_date = today) | 1K (M1), 10K (M6), 30K (Y1) | Real-time |
| WAU | Уникальные игроки с >= 1 сессией за 7 дней | COUNT(DISTINCT player_id WHERE session_date >= today-7) | 3K (M1), 25K (M6), 80K (Y1) | Daily |
| MAU | Уникальные игроки с >= 1 сессией за 30 дней | COUNT(DISTINCT player_id WHERE session_date >= today-30) | 10K (M1), 50K (M6), 150K (Y1) | Daily |

| Session Length | Средняя длительность сессии | AVG(session_end - session_start) | 25-40 мин | Per session |

### 4.3 Метрики "Responsive World" (уникальные для Nookstead)

Эти метрики не имеют аналогов в индустрии. Они измеряют качество и влияние ключевого USP.

#### 4.3.1 NPC Memory Metrics

| Метрика | Определение | Формула | Целевое значение |
|---|---|---|---|
| **NPC Surprise Rate** | % диалогов, где NPC ссылается на конкретное действие игрока | dialogues_with_memory_ref / total_dialogues | > 40% |
| **Memory Utilization** | % извлеченных воспоминаний, реально использованных в ответе | memories_used_in_response / memories_retrieved | > 50% |
| **Memory Relevance Score** | Средняя cosine similarity извлеченных воспоминаний к контексту | AVG(cosine_sim(query, retrieved_memory)) | > 0.65 |
| **Emergence Score** | % диалогов, где игрок получает неожиданную/уникальную информацию от NPC | manual_label(surprised) OR длина_ответа_игрока > median * 1.5 | > 20% |
| **Gossip Spread** | Среднее количество NPC, "знающих" о действии игрока через 48 часов | COUNT(npcs_with_memory_about(action, 48h)) | 3-5 NPC |
| **Character Consistency** | % диалогов, где NPC остается в характере (оценка модели или ручная) | consistent_dialogues / total_dialogues | > 90% |

#### 4.3.2 Как измерять "NPC Surprise Rate"

Автоматический подсчет:

```sql
SELECT
  COUNT(*) FILTER (WHERE response_text ILIKE ANY(
    ARRAY['%вчера%', '%помню%', '%ты говорил%', '%прошлый раз%',
          '%рассказывал%', '%когда ты%', '%в тот день%']
  )) AS memory_references,
  COUNT(*) AS total_dialogues,
  ROUND(100.0 * COUNT(*) FILTER (WHERE ...) / COUNT(*), 1) AS surprise_rate
FROM npc_dialogue_turns
WHERE turn_type = 'npc_response'
  AND created_at >= NOW() - INTERVAL '7 days';
```

Более точная метрика: в prompt добавить инструкцию `"memory_used": true/false` в JSON-ответ NPC. Это позволяет точно трекать, использовал ли NPC воспоминание.

#### 4.3.3 Связь NPC-метрик с бизнес-метриками

| NPC-метрика | Гипотеза связи | Бизнес-метрика | Ожидаемая корреляция |
|---|---|---|---|
| NPC Surprise Rate > 40% | Игроки, испытавшие "memory moment", возвращаются чаще | D7 Retention | r = 0.3-0.5 |
| Gossip Spread > 3 | Если 3+ NPC знают о действиях игрока, мир ощущается "живым" | Session Length (+5 мин) | r = 0.2-0.4 |
| Character Consistency < 80% | Broken character разрушает иммерсию | D1 Retention (снижение) | r = -0.3 |
| Emergence Score > 20% | Уникальные моменты -- причина поделиться с друзьями | Viral K-factor | r = 0.4-0.6 |

### 6.3 Статистический фреймворк

**Формула размера выборки (для пропорций):**

```
n = (Z_alpha/2 + Z_beta)^2 * (p1*(1-p1) + p2*(1-p2)) / (p1 - p2)^2

Для D7 retention test:
- p1 (control) = 0.25
- p2 (treatment) = 0.30
- alpha = 0.05, Z = 1.96
- beta = 0.20, Z = 0.84

n = (1.96 + 0.84)^2 * (0.25*0.75 + 0.30*0.70) / (0.05)^2
n = 7.84 * (0.1875 + 0.2100) / 0.0025
n = 7.84 * 0.3975 / 0.0025
n ≈ 1,247 на вариант

При 500/вариант -- MDE = ~8 pp (менее чувствительный, но достаточный
для detect large effects на стадии prototype).
```

**Правила раннего останова:**

| Условие | Действие |
|---|---|
| p-value < 0.001 И N > 50% от плана | Можно остановить досрочно (winner) |
| Guardrail metric нарушен | Немедленная остановка проигравшего варианта |
| p-value > 0.5 после 50% длительности | Вероятно нет эффекта -- рассмотреть остановку |

## 7. Предиктивные модели

### 7.1 Модель предсказания оттока (Churn Prediction)

**Цель:** Предсказать вероятность ухода игрока в ближайшие 7 дней.

**Определение churn:** Игрок не заходил 7+ дней (для F2P/Hybrid) или отменил подписку.

**Входные признаки (features):**

| Категория | Признак | Тип | Обоснование |
|---|---|---|---|
| **Частота** | days_since_last_login | numeric | Самый сильный предиктор |
| **Частота** | sessions_last_7_days | numeric | Тренд активности |
| **Частота** | sessions_last_7_days / sessions_prev_7_days | ratio | Детектирует спад |
| **Длительность** | avg_session_length_last_7d | numeric | Короткие сессии = потеря интереса |
| **Длительность** | session_length_trend (slope) | numeric | Снижающийся тренд = риск |
| **Вовлеченность** | npc_dialogues_last_7d | numeric | Снижение NPC-взаимодействий |
| **Вовлеченность** | unique_npcs_talked_last_7d | numeric | Разнообразие NPC |
| **Вовлеченность** | crops_planted_last_7d | numeric | Farming activity |
| **Вовлеченность** | quests_completed_last_7d | numeric | Quest engagement |
| **Прогресс** | reputation_level | numeric | High level = invested |
| **Прогресс** | days_since_registration | numeric | Tenure |
| **Прогресс** | npc_friends_count | numeric | Social bonds |
| **Социальное** | guild_member | boolean | Guild members churn less |
| **Социальное** | friends_active_last_7d | numeric | If friends leave, player follows |
| **Качество** | avg_npc_dialogue_turns | numeric | Short dialogues = disappointed |
| **Качество** | avg_dialogue_latency | numeric | High latency = poor experience |
| **Платеж** | is_subscriber | boolean | Subscribers churn less |
| **Платеж** | days_since_last_purchase | numeric | Purchase momentum |

**Архитектура модели:**

```
Алгоритм: Gradient Boosted Trees (XGBoost/LightGBM)
Обоснование: хорошо работает с табличными данными, интерпретируемый,
             не требует GPU, быстрый inference.

Training: ежедневно на данных за последние 90 дней
Inference: ежедневно для всех активных игроков
Output: churn_probability (0-1), churn_risk_tier (low/medium/high)

Пороги:
  Low risk:    P(churn) < 0.20
  Medium risk: 0.20 <= P(churn) < 0.50
  High risk:   P(churn) >= 0.50
```

**Интервенции по уровню риска:**

| Риск | Действие | Канал | Стоимость |
|---|---|---|---|
| High | NPC отправляет персонализированное письмо в почтовый ящик | In-game mail | $0.003 (Haiku) |
| High | Push-уведомление "Лена скучает по тебе" | Push | $0.00 |
| High | Скидка 30% на подписку | Email / In-game | Revenue loss |
| Medium | Ежедневные квесты с увеличенной наградой | In-game | $0.00 |
| Medium | NPC упоминает игрока при следующем входе | In-game | $0.003 |
| Low | Стандартный опыт | -- | $0.00 |

**Ожидаемые метрики модели:**

| Метрика | Целевое | Минимально допустимое |
|---|---|---|
| AUC-ROC | > 0.85 | > 0.75 |
| Precision (high risk) | > 0.60 | > 0.50 |
| Recall (high risk) | > 0.70 | > 0.60 |
| False Positive Rate | < 0.15 | < 0.25 |

### 7.2 Модель предсказания LTV (Whale Prediction)

**Цель:** По поведению в первые 3 дня предсказать LTV игрока.

**Ранние сигналы высокого LTV:**

| Сигнал | Корреляция с LTV | Обоснование |
|---|---|---|
| Длина первой сессии > 40 мин | r = 0.35 | Высокая начальная вовлеченность |
| NPC диалогов в D1 > 3 | r = 0.30 | Заинтересован в core USP |
| Вернулся на D2 | r = 0.45 | D1 retention -- лучший ранний предиктор |
| Вернулся 3 из первых 3 дней | r = 0.55 | Consistency -- key signal |
| Посадил > 5 культур в D1-D3 | r = 0.25 | Engaged with core loop |
| Посетил 4+ зоны | r = 0.20 | Explorer behavior |
| Написал free text (не quick reply) | r = 0.30 | Invested in NPC interaction |
| Покупка в первые 72 часа | r = 0.70 | Strongest predictor |

**Модель:**

```
Алгоритм: XGBoost Regression
Target: revenue_90d (доход за 90 дней)
Features: поведение D1-D3 (см. выше)
Training: после накопления 90-дневных когорт
Update: ежемесячно
```

**Применение:**

- High-LTV игроки -> premium onboarding, VIP support, exclusive NPC content
- Low-LTV at-risk -> simplified experience, stronger tutorials, push focus

### 7.3 Модель качества NPC-диалогов

**Цель:** Предсказать, будет ли ответ NPC "хорошим", до отправки игроку.

**Определение "хорошего" ответа:**

```
quality_score = w1 * in_character       (0/1, from classifier)
              + w2 * memory_used        (0/1, from JSON flag)
              + w3 * appropriate_length  (0/1, 20-150 chars)
              + w4 * player_responded    (0/1, did player continue dialogue)
              + w5 * not_generic         (0/1, from classifier)

Weights: w1=0.30, w2=0.25, w3=0.10, w4=0.20, w5=0.15
```

**Применение:**
- Мониторинг: daily quality_score по NPC, по prompt-варианту
- Алерт: если quality_score для NPC < 0.60 -- ревизия Seed Persona
- A/B-тесты: сравнение prompt-вариантов по quality_score

### 7.4 Модель оптимизации LLM-расходов

**Цель:** Предсказать LLM-расход на игрока на основе его поведения и оптимизировать.

```
predicted_llm_cost_daily(player) =
    base_cost                                    # $0.005 (plan share, reflection share)
  + dialogue_count_predicted * cost_per_dialogue  # 1.5 * $0.04 = $0.06
  + is_subscriber * bonus_dialogue_cost          # 0.3 * $0.02 = $0.006

Optimization levers:
1. Cache common greetings (saves 20-30% per dialogue)
2. Use Haiku for non-subscriber dialogues (saves 80% per turn)
3. Reduce memory retrieval count for low-importance dialogues
4. Batch reflections during off-peak hours
```

**Тирование модели по сегменту:**

| Сегмент | Модель для диалога | Memories в контексте | Ожидаемый cost/turn |
|---|---|---|---|
| Subscriber + Deep session | Sonnet | 10 | $0.007 |
| Subscriber + Quick check | Sonnet | 5 | $0.005 |
| Non-subscriber | Haiku | 5 | $0.001 |
| Tutorial (D1) | Sonnet (investment) | 0 (cold start) | $0.005 |

### B. SQL-запросы для ключевых метрик

**D7 Retention:**

```sql
SELECT
  DATE(first_login) AS cohort_date,
  COUNT(DISTINCT p.id) AS cohort_size,
  COUNT(DISTINCT CASE
    WHEN EXISTS (
      SELECT 1 FROM telemetry_events te
      WHERE te.player_id = p.id
        AND te.event_name = 'session_start'
        AND DATE(te.created_at) BETWEEN DATE(p.created_at) + 7
                                     AND DATE(p.created_at) + 7
    ) THEN p.id
  END) AS retained_d7,
  ROUND(100.0 *
    COUNT(DISTINCT CASE WHEN EXISTS (...) THEN p.id END)
    / COUNT(DISTINCT p.id), 1
  ) AS retention_d7_pct
FROM players p
WHERE p.created_at >= NOW() - INTERVAL '30 days'
GROUP BY DATE(first_login)
ORDER BY cohort_date;
```

**LLM Cost per DAU:**

```sql
SELECT
  DATE(created_at) AS date,
  SUM(cost_usd) AS total_llm_cost,
  COUNT(DISTINCT player_id) FILTER (WHERE player_id IS NOT NULL) AS dau_with_dialogue,
  (SELECT COUNT(DISTINCT player_id) FROM telemetry_events
   WHERE event_name = 'session_start'
     AND DATE(created_at) = DATE(llm_cost_log.created_at)) AS total_dau,
  ROUND(SUM(cost_usd) / NULLIF(total_dau, 0), 4) AS cost_per_dau
FROM llm_cost_log
WHERE created_at >= NOW() - INTERVAL '30 days'
GROUP BY DATE(created_at)
ORDER BY date;
```

### C. Формула расчета LTV

```
LTV = ARPU * (1 / monthly_churn_rate)

Для Hybrid модели:
LTV_subscriber = ($10 purchase + $6/month * 1/0.08) = $10 + $75 = $85
LTV_non_subscriber = $10 + $1/month * 1/0.25 = $10 + $4 = $14
Blended_LTV = 0.30 * $85 + 0.70 * $14 = $35.30

С учетом LLM costs:
Net_LTV_sub = $85 - ($2/month * 12.5 months) = $85 - $25 = $60
Net_LTV_non_sub = $14 - ($1/month * 4 months) = $14 - $4 = $10
Net_Blended = 0.30 * $60 + 0.70 * $10 = $25
```

### D. Глоссарий метрик

| Сокращение | Расшифровка | Определение |
|---|---|---|
| DAU | Daily Active Users | Уникальные игроки с >= 1 сессией за день |
| MAU | Monthly Active Users | Уникальные игроки с >= 1 сессией за 30 дней |
| ARPU | Average Revenue Per User | Выручка / MAU |
| ARPPU | Average Revenue Per Paying User | Выручка / Paying Users |
| LTV | Lifetime Value | Суммарный доход от одного игрока за все время |
| MRR | Monthly Recurring Revenue | Ежемесячная рекуррентная выручка (подписки) |
| CAC | Customer Acquisition Cost | Стоимость привлечения одного игрока |
| D1/D7/D30 | Day-N Retention | % игроков, вернувшихся на N-й день после регистрации |
| NPS | Net Promoter Score | Индекс лояльности (промоутеры - критики) |
| MDE | Minimum Detectable Effect | Минимальный эффект, который может обнаружить A/B-тест |
| AUC-ROC | Area Under ROC Curve | Метрика качества бинарного классификатора (0.5 = случайный, 1.0 = идеальный) |
