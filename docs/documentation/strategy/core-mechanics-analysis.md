## 1. Анализ Core Loop

### 1.1 Философия: почему Nookstead -- не Stardew Valley

В Stardew Valley core loop -- это **оптимизация фермы**: посадил-полил-собрал-продал-расширился. NPC -- декорация, которая мотивирует ровно до момента, пока не увидишь все скриптовые реплики (~40 часов). В Animal Crossing core loop -- **декорирование**: расставил мебель-поменял одежду-порадовался. NPC милые, но взаимозаменяемые.

В Nookstead core loop -- это **ритуал принадлежности к сообществу**. Каждое утро ты заходишь в маленький городок, где тебя знают. Пекарь спрашивает, как вчерашние дела. Садовница замечает, что ты посадил клубнику. Торговец ворчит, что ты третий день подряд несёшь помидоры. Ты не "играешь" -- ты *живёшь* в этом мире.

**Ключевое отличие от конкурентов: действия игрока -- это не самоцель, а контекст для реакции мира.**

Фермерство нужно не ради монет, а чтобы Марко сказал: "О, клубника! У меня есть рецепт пирога..." Продажа на рынке нужна не ради экономики, а чтобы торговец заворчал: "Опять помидоры? Весь город несёт помидоры!" Декорирование участка нужно, чтобы Лена пришла посмотреть и похвалила.

### 1.2 Утренняя сессия (быстрый заход, 5-10 минут)

Игрок открывает браузер утром, заходит в игру. Цель дизайна -- дать **ощущение "проверил, всё хорошо"** за 5 минут.

```
Мин 0-1:  Загрузка. Ты на своём участке.
          УВЕДОМЛЕНИЕ: "Пока тебя не было..."
          - "Лена заходила, но не застала тебя."
          - "Марко оставил записку в почтовом ящике."
          - "Помидоры выросли!"
          Эмоция: тепло, "мне тут рады", любопытство.

Мин 1-3:  УТРЕННИЙ РИТУАЛ ФЕРМЫ.
          Полить грядки (5 грядок = 5 кликов, 30 секунд).
          Собрать готовый урожай (если есть).
          Это НЕ работа, это ритуал. Как утренний кофе.
          Визуал: приятные звуки воды, ростки подрастают с анимацией.

Мин 3-4:  Проверить почтовый ящик.
          Письмо от Марко: "Привет! Вчера пёк новый хлеб, подумал о тебе --
          ты же говорил, что любишь с изюмом? Зайди, дам попробовать!"
          Хук для расширенной сессии.

Мин 4-5:  Посмотреть доску объявлений (если заходишь в город).
          3 ежедневных задания: "Принести 5 ягод Лене", "Поговорить с Олегом",
          "Продать любые 3 предмета на рынке."
          Можно сделать сейчас или вечером.
```

**Что даёт быстрый заход:**
- Урожай полит (прогресс не стоит)
- Прочитал уведомления (мир жил без тебя!)
- Получил хук для следующей сессии
- Ощущение: "Я часть этого мира"

**5-минутные быстрые задачи (Quick Tasks):**

| Задача | Время | Награда | NPC-контекст |
|--------|-------|---------|-------------|
| Полить грядки | 30 сек | Прогресс роста | Степан замечает ухоженность |
| Собрать урожай | 15 сек | Предметы в инвентаре | Торговец интересуется, что принесёшь |
| Проверить почту | 20 сек | Информация + хуки | NPC пишут персонализированные письма |
| Сбор ягод/грибов у дома | 1-2 мин | 3-5 ресурсов | Лена комментирует, если дикие цветы |
| Ежедневное задание (простое) | 2-3 мин | Монеты + репутация | NPC-реакция на выполнение |

**Как игра вознаграждает быстрый заход:**
1. **Уведомления о жизни мира** -- ты узнаёшь, что мир жил без тебя (NPC приходили, что-то случилось). Это отличает Nookstead от Character.AI, где персонаж "спит", пока ты не онлайн.
2. **Прогресс за минимальные действия** -- полил 5 грядок = прогресс на весь день. Не нужно часовой сессии.
3. **Хук для следующего визита** -- письмо от NPC с обещанием ("зайди завтра, будет сюрприз") создаёт мягкую причину вернуться.

### 1.3 Расширенная сессия (30-60 минут)

Естественный поток: **Ферма -> Город -> Социал -> Ферма**

```
Мин 0-5:    ФЕРМА (утренний ритуал)
             Полить, собрать, посадить новое.
             Решить: что продать, что подарить, что оставить для крафта.

Мин 5-7:    ПЕРЕХОД В ГОРОД
             Анимация перехода (5-12 сек). Чувство "я иду в люди".
             Первый контакт: NPC идёт мимо, машет рукой.

Мин 7-12:   ТОРГОВЛЯ
             Рынок: продать урожай. NPC-торговец комментирует.
             "Клубника! Марко спрашивал, нет ли у кого клубники -- беги к нему!"
             RESPONSIVE MOMENT #1: торговец связывает действия игрока с NPC.
             Купить семена/инструменты на заработанные монеты.

Мин 12-18:  ПЕРВЫЙ ДИАЛОГ (ключевой)
             Зайти к Марко (по наводке торговца или по своему желанию).
             Марко: "О, у тебя клубника?! Я как раз хотел попробовать
             новый рецепт пирога! Дашь немного?"
             RESPONSIVE MOMENT #2: NPC реагирует на конкретные действия.
             Выбор: подарить клубнику (усиливает отношения) или оставить.
             Разговор на 3-5 реплик. Марко рассказывает о своих делах.

Мин 18-24:  ВТОРОЙ ДИАЛОГ или ИССЛЕДОВАНИЕ
             Вариант A: Зайти к Лене в парк. Она замечает что-то новое.
             "Я видела, что ты посадил цветы у дома. Какие красивые!
             Знаешь, у меня есть семена лаванды -- хочешь?"
             RESPONSIVE MOMENT #3: NPC замечает изменения на участке.

             Вариант B: Исследование леса. Сбор ресурсов. Нарваться на
             диалог NPC-NPC (Марко и Ира в кафе обсуждают):
             "А тот новенький -- нормальный. Клубнику мне принёс!"
             RESPONSIVE MOMENT #4: NPC обсуждает игрока за его спиной.

Мин 24-28:  СВОБОДНОЕ ВРЕМЯ
             Декорирование участка / рыбалка / квест от доски объявлений.
             Расслабленный темп. Нет давления.

Мин 28-30:  ЗАВЕРШЕНИЕ
             Возвращение домой. Проверка прогресса грядок.
             УВЕДОМЛЕНИЕ: "Марко попросил кого-то о помощи с крышей.
             Если хочешь помочь -- зайди завтра."
             Хук для следующей сессии.
```

**Где возникают "Responsive Moments" (точки реакции мира):**

| Момент | Триггер | Пример | Вероятность за сессию |
|--------|---------|--------|----------------------|
| NPC комментирует продажу | Продажа на рынке | "Опять помидоры? Рынок завален!" | 100% (если продал) |
| NPC просит конкретный ресурс | Наличие ресурса в инвентаре | "У тебя клубника? Мне бы для пирога..." | 30-50% |
| NPC замечает изменения | Посадка/декор на участке | "Видела твои цветы! Красота!" | 1 раз / 2-3 дня |
| NPC обсуждает игрока | Накопленные действия | "Тот фермер -- трудяга, каждый день на рынке" | 1 раз / сессию |
| NPC вспоминает прошлое | Повторный визит | "Помнишь, ты рассказывал про...?" | 100% (при 2+ визитах) |
| NPC приносит подарок | Глубокие отношения | "Вот, испёк для тебя булочку" | 1 раз / неделю |

**Что мотивирует выходить за пределы участка:**
1. **NPC как причина** -- письма, слухи, подсказки торговца. "Иди к Марко -- он спрашивал про тебя."
2. **Экономика** -- продать урожай можно только на рынке. Семена купить -- только в магазине.
3. **Социальное любопытство** -- "Что NPC обсуждают сегодня?" Бар вечером -- epicenter сплетен.
4. **Ежедневные задания** -- доска объявлений даёт конкретные цели в городе.
5. **Другие игроки** -- увидеть, что делает сосед, похвастаться урожаем.

### 1.4 Поздняя игра (100+ часов)

**Проблема:** В Stardew Valley после 100 часов все диалоги прочитаны, все NPC "прокачаны", оптимальная ферма построена. В Animal Crossing после 200 часов все жители -- милые, но пустые куклы. Как Nookstead сохраняет интерес?

**Решение 1: AI-контент по природе бесконечен**

Скриптовые игры имеют конечный контент. Nookstead -- нет. Каждый разговор уникален, потому что контекст постоянно меняется:
- Сезон сменился -- NPC обсуждают новые культуры, погоду, события
- Другие игроки на сервере совершают действия -- NPC упоминают их
- NPC формируют новые рефлексии -- их характер *развивается*
- Конфликты между NPC создают эмерджентные сюжеты

**Решение 2: Эмерджентные события**

Система агрегирует состояние NPC и создаёт нескриптовые события:

| Условие | Событие | Вовлечение игрока |
|---------|---------|-------------------|
| Среднее "счастье" NPC > 8/10 | Спонтанный праздник в парке | Участие, подарки, мини-игры |
| 3+ NPC "обеспокоены" | Городское собрание у мэра | Голосование, помощь, обсуждение |
| 2 NPC в конфликте | Город "принимает стороны" | Игрок может помирить или подлить масла |
| Сезон закончился | Фестиваль урожая | Конкурсы, NPC оценивают продукты игроков |
| Игрок не был 3+ дней | NPC "волнуются" | Марко: "Ты где пропадал? Я волновался!" |

**Решение 3: Прогрессия глубины отношений**

После 100 часов отношения с NPC достигают уровня Close Friend. Это *не конец прогрессии* -- это начало нового слоя:
- NPC делятся личными тайнами (вложенные AI-сюжеты)
- NPC просят сложной помощи (квесты, требующие ресурсов и времени)
- NPC начинают конфликтовать друг с другом из-за игрока (ревность, конкуренция)
- NPC дарят уникальные предметы, недоступные в магазине

**Решение 4: Серверные уникальные истории**

Каждый сервер развивается уникально. На одном сервере Марко и Сара помирились и открыли совместное кафе. На другом -- Марко закрыл пекарню и стал рыбаком. Игроки обсуждают это в Discord, создавая мета-контент.

**Решение 5: Инъекция событий (контроль со стороны разработчиков)**

Раз в 2-4 недели разработчики добавляют "событийные триггеры" -- новые факты в память NPC:
- "В лесу видели странный свет" -- NPC начинают обсуждать, игроки расследуют
- "Приехал новый житель" -- новый NPC с уникальной персоной
- "Рынок затопило" -- временное событие, NPC перестраиваются

Это стоит $0 в разработке (просто новые записи в базе данных), но создаёт свежесть.

## 2. Механика "Отзывчивого мира"

### 2.1 Система наблюдения за действиями игрока (Player Action Observation System)

Это ядро уникальности Nookstead. NPC не просто отвечают на вопросы -- они *замечают*, что ты делаешь, и *реагируют* на это. Технически это event-driven система, где действия игрока превращаются в записи в памяти NPC.

#### Полная таблица наблюдаемых действий

| Действие игрока | Событие для NPC | Пример реакции | Технический триггер | Задержка реакции | Стоимость LLM |
|----------------|-----------------|----------------|---------------------|------------------|---------------|
| Посадил конкретную культуру | `player_planted:{crop}` | Марко: "Клубника? Есть рецепт пирога!" | `on_crop_planted` event | Следующий визит | $0 (inject в память) |
| Собрал урожай | `player_harvested:{crop}:{qty}` | Степан: "Хороший урожай! Завтра удобрения привезу" | `on_crop_harvested` event | Следующий визит | $0 |
| Продал на рынке | `player_sold:{item}:{qty}:{price}` | Торговец: "Третий день помидоры? Попробуй тыквы" | `on_item_sold` + aggregate | В момент продажи | $0.002 (Haiku) |
| Продаёт одно и то же 3+ дней | `player_pattern:repeat_sell:{item}` | Другой NPC: "Слышал, ты стал помидорным королём!" | Аггрегация sell events | 1-2 дня | $0 (inject) |
| Посетил NPC часто | `player_visits:{npc}:{count}` | NPC: "Мой самый частый гость! Есть кое-что для тебя" | `visit_count` > threshold | При достижении порога | $0 (inject) |
| Не заходил 3+ дней | `player_absent:{days}` | Марко: "Куда пропал? Волновался!" | `absence_detection` cron | При возвращении | $0 (inject) |
| Помог NPC с квестом | `player_helped:{npc}:{quest}` | Другие NPC: сплетня "Он починил крышу Марко!" | `on_quest_complete` event | 1-2 часа (gossip spread) | $0 (inject) |
| Подарил предмет NPC | `player_gifted:{npc}:{item}` | NPC запоминает, благодарит через день | `on_gift` event | Мгновенно + follow-up | $0.002 (importance) |
| Посадил цветы на участке | `homestead_changed:flowers` | Лена: "Видела цветы! Могу дать семена лаванды" | `on_homestead_change` event | 1-3 дня | $0 (inject) |
| Расширил участок | `homestead_changed:expansion` | Мэр: "Твой участок растёт! Скоро лучший в городе" | `on_homestead_change` event | Следующий визит | $0 (inject) |
| Начал новую культуру | `player_first_crop:{crop}` | Степан: "Первый раз сажаешь кукурузу? Могу дать совет" | First-time detection | Следующий визит | $0 (inject) |
| Поговорил с NPC на тему | `player_discussed:{topic}` | Другой NPC: "Марко рассказывал, ты интересуешься..." | Memory cross-reference | При gossip | $0 |
| Купил семена в магазине | `player_bought:{item}` | NPC: "Видел тебя в магазине. Тыквенные семена? Мудрый выбор" | `on_purchase` event | 0-1 день | $0 (inject) |

#### Ключевой принцип: большинство "наблюдений" стоят $0

Реакции NPC НЕ требуют дополнительных LLM-вызовов. Механизм:

1. Действие игрока создаёт **событие** (event)
2. Событие форматируется в текстовую **запись памяти** по шаблону
3. Запись добавляется в `memory_stream` соответствующего NPC
4. При следующем диалоге NPC memory retrieval подтягивает эту запись
5. LLM естественно использует её в разговоре

**Пример:**

```
Событие:  on_crop_planted(player="CoolFarmer", crop="strawberry")
Шаблон:   "Я видел(а), что {player} посадил(а) {crop} на своём участке."
Запись:   "Я видел, что CoolFarmer посадил клубнику на своём участке."
Важность: 5 (средняя -- не тривиально, но не грандиозно)
NPC:      Лена (садовник), Марко (пекарь -- любит ягоды)
```

Когда CoolFarmer в следующий раз заговорит с Марко, retrieval подтянет эту запись (если она релевантна запросу), и LLM сам решит, как на неё отреагировать.

**Почему это работает:**
- **$0 cost** для "наблюдения" -- только шаблонная запись в БД
- **Естественная интеграция** -- LLM сам решает, когда упомянуть
- **Масштабируемость** -- сколько угодно наблюдений без роста LLM-расходов
- **Непредсказуемость** -- игрок не знает, что NPC заметил; каждый раз сюрприз

#### Какие NPC "видят" какие действия

Не каждый NPC видит всё. Наблюдения распределяются по **зонам осведомлённости**:

| NPC | Видит на участке игрока | Видит в городе | Узнаёт от других NPC |
|-----|------------------------|----------------|---------------------|
| Лена (садовник) | Цветы, декор двора, состояние грядок | Кого встречает в парке | Сплетни от Иры |
| Марко (пекарь) | Ничего (не приходит на участок) | Покупки, продажи, визиты к нему | От Иры, от торговца |
| Степан (агроном) | Все фермерские действия | Что продаёт на рынке | Мало общается (замкнутый) |
| Ира (кафе) | Ничего | ВСЁ -- кафе = центр городской жизни | Все NPC приходят в кафе |
| Торговец | Ничего | Все покупки и продажи | От других торговцев |
| Виктор (мэр) | Расширение участка, здания | Квесты, доска объявлений | От всех |

**Механизм gossip (сплетни):** Когда NPC-NPC проводят спонтанный разговор, информация о действиях игрока может "перетечь" от одного NPC к другому. Технически -- запись в память второго NPC: "Ира рассказала, что CoolFarmer принёс Марко клубнику."

### 2.2 NPC-инициированные взаимодействия

В большинстве игр игрок всегда подходит к NPC первым. В Nookstead NPC иногда сами ищут контакта. Это ключевое отличие -- мир не ждёт тебя, а *живёт*.

#### Типы NPC-инициированных взаимодействий

| Тип | Триггер | Реализация | Частота |
|-----|---------|------------|---------|
| **NPC приходит на участок** | Уровень отношений >= Casual Friend | NPC появляется у забора, ждёт 5 мин | 1 раз / 3-5 дней |
| **Письмо в почтовом ящике** | Любое событие, которое NPC хочет обсудить | Текст генерируется Haiku по шаблону | 1-2 / день |
| **Сплетня при подслушивании** | Накоплена информация о действиях игрока | NPC-NPC разговор, где упоминают игрока | 1 раз / сессию |
| **Подарок от NPC** | Уровень >= Friend, NPC "подумал" о подарке в рефлексии | Предмет в почтовом ящике + записка | 1 раз / неделю |
| **NPC подходит в городе** | NPC рядом + триггер (нужна помощь, хочет рассказать) | NPC меняет маршрут, идёт к игроку | 1 раз / 2-3 сессии |
| **NPC зовёт на мероприятие** | Приближается фестиваль / NPC планирует что-то | Письмо: "Приходи завтра в парк в 18:00!" | При событиях |

#### Детали: NPC приходит на участок

**Когда:** NPC отклоняется от своего дневного плана, если в рефлексии появляется мысль типа: "Хочу проведать CoolFarmer -- давно не видел."

**Как:**
1. Рефлексия создаёт запись: `want_to_visit:CoolFarmer`
2. При генерации следующего дневного плана NPC включает "посетить CoolFarmer" в расписание
3. NPC приходит к забору участка, встаёт у ворот
4. Если игрок онлайн -- NPC ждёт 5 минут (виден на мини-карте)
5. Если игрок подходит -- начинается диалог
6. Если игрок не подходит / не онлайн -- NPC уходит, оставляет запись: "Заходил к CoolFarmer, не застал"
7. При следующей встрече NPC упоминает: "Я заходил к тебе, но тебя не было"

**Дизайнерская ценность:** Это мощнейший retention-хук. В Animal Crossing жители иногда стоят у твоего дома -- это один из самых любимых моментов игроков. В Nookstead это усилено контекстом: NPC приходит по *причине*, связанной с историей взаимодействий.

#### Детали: Письма в почтовом ящике

Формат письма (генерируется Haiku по шаблону, стоимость ~$0.001):

```
От: Марко
Тема: Клубничный пирог!

Привет, {player}!

Помнишь, ты дал мне клубнику на прошлой неделе?
Я попробовал новый рецепт -- получилось! Зайди ко мне,
дам попробовать кусочек.

-- Марко

P.S. Если у тебя будет ещё клубника, принеси --
хочу попробовать вариант с шоколадом!
```

**Тригеры для писем:**
- NPC обработал рефлексию, в которой есть мысль о игроке
- Игрок подарил предмет (благодарственное письмо через день)
- NPC нужна помощь (органический квест)
- Приближается событие (приглашение)
- Игрок не был 3+ дней (беспокойство)

#### Детали: Сплетни

Самый вирусный момент в Nookstead. Игрок гуляет по городу и *случайно* слышит, как два NPC обсуждают его.

**Техническая реализация:**
1. Два NPC оказываются рядом (по расписанию или случайно)
2. Система проверяет: есть ли у обоих NPC воспоминания об одном игроке?
3. Если да -- с вероятностью 30% запускается NPC-NPC разговор на эту тему
4. Разговор генерируется Haiku (3-5 строк, ~$0.002)
5. Текст отображается как "пузыри" над NPC
6. Игрок видит это, если находится в радиусе 10 тайлов

**Пример:**
```
Марко: "Этот CoolFarmer -- хороший человек. Клубнику мне носит."
Ира: "Да? Ко мне тоже заходит. Всегда вежливый."
Марко: "Может, пригласим его на ужин в пятницу?"
```

**Почему это работает:**
- Создаёт ощущение "мир говорит обо мне"
- Идеально для скриншотов/клипов (вирусный потенциал)
- Связывает NPC между собой (они не изолированные чат-боты)
- Стоит всего ~$0.002 за сплетню

### 2.3 Система распространения информации между NPC (Gossip Propagation)

Информация о действиях игрока не мгновенно доступна всем NPC. Она *распространяется* через социальную сеть города, как в реальном маленьком городке.

```
Действие игрока
     │
     ▼
NPC-свидетель (видел лично)
     │
     ▼ (при NPC-NPC разговоре)
NPC-первый-круг (услышал от свидетеля)
     │
     ▼ (при NPC-NPC разговоре, с меньшей вероятностью)
NPC-второй-круг (слышал "через одного")
```

**Правила распространения:**

| Путь | Вероятность передачи | Искажение | Пример |
|------|---------------------|-----------|--------|
| Свидетель → Первый круг | 60% при разговоре | Минимальное | "CoolFarmer принёс клубнику Марко" |
| Первый круг → Второй круг | 30% при разговоре | Среднее | "Говорят, CoolFarmer дружит с Марко" |
| Второй круг → Третий | 10% | Сильное | "Тот новенький, кажется, хороший парень" |

**Дизайнерская ценность:**
- Создаёт временнУю задержку -- NPC не все сразу всё знают
- Добавляет непредсказуемость -- разные NPC знают разное
- Создаёт моменты открытия -- "Откуда ты знаешь, что я выращиваю клубнику?!" -- "Марко рассказал!"

## 3. Механика фермерства (MVP)

### 3.1 Философия: фермерство как ритуал и контекст, а не как система

В Stardew Valley фермерство -- это *работа*: оптимизация сетки, min-maxing урожая, спринклерные системы, расчёт ROI по культурам. Это вовлекает "геймерский" сегмент, но отпугивает casual-аудиторию.

В Nookstead фермерство -- это **утренний ритуал** (как полить цветы на подоконнике) и **контекст для реакции мира** (NPC замечают, что ты выращиваешь). Глубина приходит не от оптимизации фермы, а от реакции NPC на твой выбор.

### 3.2 Дизайн культур для MVP (5 культур)

| Культура | Сезон | Время роста (реальное) | Полив | Цена продажи | Ключевая NPC-связь | Дизайнерская роль |
|----------|-------|----------------------|-------|-------------|--------------------|--------------------|
| **Редис** | Весна/Лето | 12 часов | 1x/день | 15 монет | Степан: "Хорошее начало для новичка!" | Первый урожай -- instant gratification |
| **Картофель** | Весна | 2 дня | 1x/день | 25 монет | Ира: "Из картошки суп сварю!" | Стабильный доход, простой уход |
| **Клубника** | Весна/Лето | 3 дня | 2x/день | 40 монет | Марко: "Для пирога -- идеально!" | Требует больше ухода, но NPC любят |
| **Помидоры** | Лето | 4 дня | 2x/день | 30 монет | Торговец: "Опять помидоры?!" | Массовый продукт, динамика рынка |
| **Тыква** | Осень | 6 дней | 2x/день | 60 монет | Лена: "Какая красивая тыква!" | Долгое ожидание -- большая награда |

#### Почему именно эти 5 культур

1. **Редис** -- reward за 12 часов. Посадил утром -- вечером урожай. Или посадил вечером -- утром урожай. Это *не* заставляет ждать дни. Первый magic moment фермерства: "Я что-то вырастил!"

2. **Картофель** -- стабильный средний класс. 2 дня = первый "настоящий" цикл ожидания. Низкий уход (1x полив). Связь с Ирой (кафе) -- контекст для еды/рецептов.

3. **Клубника** -- "усиленный выбор". Дороже, но требует 2x полива. Ключевая связь с Марко (пекарь) -- именно клубника запускает самый яркий NPC-responsive moment ("Для пирога!"). Это *дизайнерский крючок*: клубника = лучший способ подружиться с Марко.

4. **Помидоры** -- массовая культура. Много дешёвых помидоров = триггер для торговца ("рынок завален помидорами!"). Учит игрока разнообразию. Если все выращивают помидоры -- NPC реагируют совместно.

5. **Тыква** -- "терпеливый игрок". 6 дней -- долгое ожидание. Но награда максимальная. NPC впечатлены ("какая огромная тыква!"). Фестиваль тыкв осенью. Это для игроков, которые вкладываются в долгосрочную стратегию.

#### Фазы роста (визуально)

Каждая культура проходит 5 визуальных стадий (спрайты из Modern Farm):

```
[Семя] → [Росток] → [Рост] → [Зрелость] → [Урожай] → (если не собрал 2 дня) [Увядание]
```

| Стадия | Длительность (% от общего времени) | Визуал | Взаимодействие |
|--------|-----------------------------------|--------|---------------|
| Семя | 0-15% | Пустая грядка с меткой | Полив |
| Росток | 15-35% | Маленький зелёный росток | Полив |
| Рост | 35-65% | Растение средней высоты | Полив |
| Зрелость | 65-90% | Полноразмерное с плодами | Полив |
| Урожай | 90-100% | Плоды ярко-подсвечены, "мерцают" | СБОР (клик) |
| Увядание | +48 часов после урожая | Потемневшее, поникшее | Убрать (урожай потерян) |

**Анимация сбора:** Приятный "pop" эффект + звук + числа урожая всплывают. Аналог -- Animal Crossing при встряхивании дерева. Момент должен быть *сенсорно приятным*.

### 3.3 Прогрессия фермы

#### Начальный участок (3x3 грядок)

```
+---------------------------+
|  [Дом]                    |
|                           |
|  [G][G][G]   [Почтовый   |
|  [G][G][G]    ящик]      |
|  [G][G][G]               |
|                [Забор/    |
|                 выход]    |
+---------------------------+
G = грядка (3x3 = 9 грядок)
```

**9 грядок** -- достаточно для утреннего ритуала (9 кликов полива = ~30 секунд) и для получения значимого урожая.

#### Расширение фермы

| Уровень | Размер | Стоимость | Время разблокировки | Примечание |
|---------|--------|-----------|--------------------|-----------|
| Начальный | 3x3 (9 грядок) | Бесплатно | Старт игры | Достаточно для первых 1-2 недель |
| Уровень 2 | 4x4 (16 грядок) | 500 монет | ~1 неделя | Мэр: "Участок растёт!" |
| Уровень 3 | 5x5 (25 грядок) | 1500 монет | ~3 недели | Степан: "Серьёзный фермер!" |
| Уровень 4 | 6x6 (36 грядок) | 4000 монет | ~6 недель | Лена: "Лучший огород в городе!" |

**NPC-реакция на расширение:** Каждое расширение -- событие. Мэр поздравляет. NPC комментируют. Это не просто "купил апгрейд" -- мир *замечает* твой рост.

#### Инструменты (3 уровня для MVP)

| Инструмент | Уровень 1 | Уровень 2 | Уровень 3 |
|------------|-----------|-----------|-----------|
| **Лейка** | Поливает 1 грядку | Поливает 3 грядки в ряд | Поливает 3x3 зону |
| **Мотыга** | Вскапывает 1 грядку | Вскапывает 3 в ряд | 3x3 зону |
| **Серп** | Собирает 1 урожай | 3 в ряд | 3x3 зону |

**Цены:** Уровень 2 = 300 монет, Уровень 3 = 1200 монет (у NPC-механика Егора).
**NPC-связь:** Егор продаёт улучшения. Диалог: "Новая лейка? Раньше я делал такие для деда-фермера. Хороший инструмент, береги!"

#### Время до "rewarding loop" (цель: первый урожай за 12-24 реальных часа)

```
Час 0:    Получил участок. Степан даёт 3 пакетика семян редиса бесплатно.
Час 0-5:  Вскопал 3 грядки, посадил, полил. (1 минута работы)
          Степан: "Хорошее начало! Приходи за советом."
Час 12:   Редис вырос! Собрал 3 штуки.
          MAGIC MOMENT: первый урожай. Звуки, анимация, чувство achievement.
Час 12-13: Продал на рынке (45 монет).
          Торговец: "Первый урожай? Поздравляю! Бери семена картошки -- хорошая цена."
          Купил 5 семян картошки (25 монет).
Час 13:   Посадил картошку. Ждёшь 2 дня.
          Марко (если зашёл): "Слышал, у тебя первый урожай! Молодец!"
Час 48:   Картошка выросла. Продал (125 монет). Можно расширить грядки.
```

**Вывод:** За 2 реальных дня игрок проходит полный цикл фермерства, получает первые деньги, слышит реакции NPC. К третьему дню может купить клубнику и запустить "Марко-пирог" quest chain.

### 3.4 Полив: ритуал, а не наказание

**Проблема SDV:** В ранней игре полив -- скучная обязанность. Если пропустил -- растения засыхают. Это создаёт FOMO и стресс.

**Решение Nookstead:**
- **Пропущенный полив НЕ убивает растения** -- рост просто замирает (пауза)
- Если вернулся через 3 дня -- растения на той же стадии, не мёртвые
- Но NPC замечает: "Твои грядки подсохли -- всё хорошо? Полей, и они оживут"
- **Дождь = автополив** -- в дождливый день поливать не нужно (бонус 25% роста)
- **Спринклер** (Phase 3) -- автоматизация для поздней игры

**Тон:** Мягкое напоминание, не наказание. Как друг, который говорит: "Эй, не забудь про цветы."

## 4. Экономический цикл

### 4.1 Источники дохода игрока

| Источник | Доход за единицу | Частота | Доступно | Примечание |
|----------|-----------------|---------|----------|-----------|
| Продажа урожая | 15-60 монет | Ежедневно | MVP | Основной доход |
| Продажа дикоросов | 5-15 монет | Ежедневно | MVP | Ягоды, грибы, цветы, ракушки |
| Ежедневные задания (доска) | 30-80 монет | 3/день | MVP | Стабильный дополнительный доход |
| Квесты от NPC | 50-200 монет | 1-3/неделю | Phase 2 | Органические квесты |
| Подарки от NPC | Предметы (не монеты) | 1/неделю | MVP | Не монетизированы напрямую |

### 4.2 Расходы игрока

| Расход | Цена | Примечание |
|--------|------|-----------|
| Семена (редис) | 5 монет | Самые дешёвые |
| Семена (картофель) | 8 монет | |
| Семена (клубника) | 15 монет | Дороже, но доходнее |
| Семена (помидоры) | 12 монет | |
| Семена (тыква) | 20 монет | Дорого, но самый прибыльный урожай |
| Расширение участка | 500-4000 монет | Milestone purchases |
| Улучшение инструмента | 300-1200 монет | У Егора |
| Подарки NPC (покупные) | 20-100 монет | Цветы, выпечка, книги из магазинов |
| Декор дома | 50-500 монет | Phase 2 |

### 4.3 NPC-специфические экономические предпочтения

**Вместо** единой рыночной цены, NPC-магазины предлагают разные цены на разные товары:

| NPC | Покупает дороже | Покупает дешевле | Продаёт |
|-----|----------------|-----------------|---------|
| **Торговец на рынке** | Всё по базовой цене | -- | Семена, базовые инструменты |
| **Марко (пекарь)** | Пшеница (+30%), клубника (+20%), яйца (+25%) | Всё остальное (-20%) | Хлеб, выпечка (для подарков) |
| **Лена (садовник)** | Цветы (+50%), редкие травы (+40%) | Овощи (-30%) | Семена цветов, горшки |
| **Ира (кафе)** | Картофель (+20%), помидоры (+15%), овощи в целом (+10%) | Сырьё (-10%) | Готовая еда (для подарков) |
| **Степан (агроном)** | Не покупает (не торговец) | -- | Удобрения, советы |

**Дизайнерская цель:** Игрок *думает*, кому продать урожай. Не min-maxing, а осмысленный выбор: "Отнесу клубнику Марко -- он заплатит больше И это усилит наши отношения." Экономика + социальная динамика переплетаются.

### 4.4 Ценообразование для MVP -- фиксированное

Динамические цены (из GDD v3) -- **вырезаны до Phase 2+**. Причина: добавляют сложность без прямой связи с core USP. Для MVP достаточно фиксированных цен + NPC-комментариев.

**Исключение:** NPC-торговец всё равно *комментирует* перенасыщение рынка, даже если цена не меняется. "Ох, третий день подряд помидоры... Ладно, возьму, но принеси что-нибудь другое завтра, а?" Это создаёт *ощущение* динамики без технической сложности.

### 4.5 Экономический баланс (математика)

**Цель:** Игрок за 7 реальных дней регулярной игры (30 мин/день) зарабатывает достаточно на первое расширение участка (500 монет).

| День | Действие | Доход | Расход | Баланс |
|------|---------|-------|--------|--------|
| 1 | Посадил 3 редиса (бесплатные семена), собрал ягоды | 0 + 15 = 15 | 0 | 115 (старт 100) |
| 1 (вечер) | Урожай редиса (12 часов), продажа | 45 | -15 (семена) | 145 |
| 2 | Полил, посадил картошку (5 шт), задание | 50 (задание) | -40 (семена) | 155 |
| 3 | Полил, ягоды, задание | 50 + 15 = 65 | 0 | 220 |
| 4 | Урожай картофеля! 5 шт = 125 монет | 125 + 50 (задание) | -60 (новые семена) | 335 |
| 5 | Полил клубнику, редис, задание | 50 + 45 (редис) | -15 | 415 |
| 6 | Задание + ягоды | 65 | 0 | 480 |
| 7 | Клубника! (3 шт = 120) + задание | 170 | -50 (семена) | 600 |

**День 7: 600 монет.** Достаточно для расширения (500) с остатком на семена.

**Вывод:** Экономика сбалансирована для первого milestone за 1 неделю. Это ощущается как хороший прогресс -- не слишком быстро (нет смысла спешить), не слишком медленно (не теряется мотивация).

## 5. Прогрессия отношений

### 5.1 Почему "без сердечек"

В Stardew Valley отношения -- это числа. Дал 2 тыквы в неделю x 8 недель = максимальная дружба. Это **гриндовая механика**, замаскированная под социальную.

В Nookstead нет видимых чисел. Игрок не знает, "сколько сердечек" у Марко. Он чувствует отношения через *поведение NPC*: как здоровается, что рассказывает, приходит ли в гости. Это ближе к реальным отношениям -- ты не знаешь "числовой уровень дружбы" с другом в реальной жизни.

### 5.2 Пять уровней отношений (AI-driven)

| Уровень | Поведение NPC | Как достигается | Примерное время | Внутренний триггер |
|---------|--------------|-----------------|-----------------|-------------------|
| **Stranger** | Вежливая дистанция. "Здравствуйте, вы новенький?" Формальное обращение. Шаблонные реплики. | Автоматически при первой встрече | Мгновенно | default |
| **Acquaintance** | Узнаёт по имени. "О, {имя}! Заходи." Чуть теплее. Quick replies более разнообразные. | 2-3 диалога с NPC | 1-2 дня | >= 3 записей в памяти |
| **Casual Friend** | Делится сплетнями. "Между нами -- Ира вчера поссорилась с Олегом..." Предлагает помощь. Обращается на "ты" (если NPC на "вы" по характеру). | 5-10 диалогов + 1-2 подарка | 1 неделя | Рефлексия содержит позитивные выводы |
| **Friend** | Скидки в магазине (10%). Приглашает домой. Рассказывает о себе (личные истории). Делает маленькие подарки. Просит о помощи (органические квесты). | 15+ диалогов + помощь с квестом | 2-3 недели | Рефлексия: "Считаю {player} другом" |
| **Close Friend** | Делится тайнами (уникальный контент). Приходит на участок. Дарит редкие предметы. Защищает репутацию игрока перед другими NPC. Уникальные диалоговые ветки. | Глубокое взаимодействие: помощь, время, внимание | 1-2 месяца | Рефлексия: "Один из самых близких людей" |

### 5.3 Механизм уровней: как AI определяет уровень

**Не числа, а рефлексии.** Каждый день NPC генерирует рефлексию. В рефлексии может появиться мысль о конкретном игроке:

```
День 1:  "Сегодня заходил новый человек -- CoolFarmer. Показался приятным."
День 3:  "CoolFarmer снова заходил. Подарил клубнику. Приятно."
День 7:  "CoolFarmer часто заходит. Считаю его хорошим знакомым. Можно поделиться сплетней."
День 14: "CoolFarmer помог починить крышу и всегда приносит клубнику. Это друг."
День 30: "CoolFarmer -- один из самых близких людей в городе. Доверяю ему."
```

**Технически:** Сервер парсит рефлексии на ключевые фразы (regex / keyword detection) и присваивает уровень отношений. Это НЕ LLM-вызов -- это pattern matching по рефлексиям, которые уже сгенерированы.

| Ключевые фразы в рефлексии | Уровень |
|----------------------------|---------|
| "новый", "незнакомец", "впервые" | Stranger |
| "знакомый", "приятный", "заходит" | Acquaintance |
| "хороший знакомый", "нравится", "приятно общаться" | Casual Friend |
| "друг", "доверяю", "помог", "важный" | Friend |
| "близкий друг", "один из самых", "дорог мне" | Close Friend |

### 5.4 Деградация отношений

Отношения могут ухудшаться. Но -- в духе cozy-дизайна -- **не наказательно, а мягко**:

| Ситуация | Последствие | Скорость |
|----------|------------|----------|
| Не заходил к NPC 2 недели | Уровень -1 (Friend → Casual Friend) | Медленно |
| Грубость в разговоре | NPC обижается, но прощает через 2-3 дня | Быстрое восстановление |
| Помог "врагу" NPC | NPC расстроен, но не разрывает отношения | Ситуативно |
| Обещал что-то и не сделал (квест) | NPC разочарован, упоминает это | Восстанавливается |

**Ключевой принцип:** Нельзя *потерять* отношения мгновенно. Деградация -- медленная, восстановление -- быстрое. Это cozy game, не Dark Souls.

### 5.5 Milestones отношений (уникальный контент)

Каждый NPC имеет уникальный контент, доступный только на определённом уровне отношений. Это основной мета-loop -- причина развивать отношения.

| NPC | Casual Friend | Friend | Close Friend |
|-----|--------------|--------|-------------|
| **Марко** | Сплетня о Саре | История о ресторане в городе | Тайна: боится банкротства, просит помощь с бизнес-планом |
| **Лена** | Совет по цветам | Показывает тайный сад | Подарок: редкие семена, которых нет в магазине |
| **Олег** | Рыбацкая байка | Учит рыбалке (мини-игра) | Рассказывает о "Старом городе" (лор) |
| **Ира** | Городские сплетни | Рецепт (крафт, Phase 3) | Предупреждает о конфликтах между NPC |
| **Виктор** | Квест от доски | Рассказывает историю города | Тайна: город основан не случайно (сюжет) |
| **Степан** | Совет по удобрениям | Даёт редкие семена | Рассказывает о "золотой культуре" |
| **Зоя** | Странные фразы | Фрагменты памяти | Ключ к главной тайне (Phase 3) |

## 6. Пейсинг сессий и хуки удержания

### 6.1 Целевой дневной пейсинг (20-40 минут)

```
┌─────────────────────────────────────────────────────────────┐
│ 0    5    10   15   20   25   30   35   40                  │
│ ├────┤────┤────┤────┤────┤────┤────┤────┤                   │
│ ФЕРМА│ ГОРОД │ СОЦИАЛ  │ СВОБОД. │ЗАВЕРШЕН│                 │
│ poliv│ рынок │ диалог1 │ квест/  │ почта  │                 │
│ сбор │ магазин│ диалог2 │ декор/  │ хук    │                 │
│      │       │ подслуш │ рыбалка │        │                 │
│ ▲    │ ▲     │ ▲       │         │ ▲      │                 │
│ ритуал│прогр. │ эмоция  │         │ причина│                 │
│      │       │         │         │ вернутся│                 │
└─────────────────────────────────────────────────────────────┘
```

**0-5 мин: Утренний ритуал (ферма)**
- Полив (30 сек)
- Сбор урожая (если готов)
- Проверка уведомлений
- *Эмоция:* рутинный комфорт, "всё на месте"

**5-15 мин: Город (продуктивная активность)**
- Продажа урожая на рынке (NPC-торговец комментирует)
- Покупка семян / инструментов
- 1-2 ежедневных задания с доски
- *Эмоция:* прогресс, "зарабатываю"

**15-25 мин: Социальная фаза (ядро Nookstead)**
- 1-2 полноценных диалога с NPC
- Подслушивание NPC-NPC разговора
- Подарок NPC
- *Эмоция:* тепло, удивление, связь

**25-35 мин: Свободная фаза**
- Квест от NPC
- Декорирование участка
- Рыбалка (мини-игра)
- Исследование леса
- *Эмоция:* расслабление, самовыражение

**35-40 мин: Завершение**
- Возвращение домой
- Проверка почты (хук для следующего дня)
- *Эмоция:* завершённость, предвкушение

### 6.2 Хуки для возврата (Return Triggers)

| Хук | Механизм | Пример | Давление |
|-----|---------|--------|---------|
| **Урожай будет готов** | Реальное время роста | "Картофель будет готов завтра утром" | Мягкое (урожай не пропадёт) |
| **NPC обещал что-то** | Запись в памяти NPC | "Марко сказал зайти завтра за пирогом" | Мягкое (NPC подождёт) |
| **Смена сезона** | Каждые 7 реальных дней | "Скоро лето -- новые культуры!" | Информационное |
| **Письмо от NPC** | Триггер из рефлексии | "Лена написала: приходи, показать кое-что" | Любопытство |
| **Незавершённый квест** | Active quest state | "Марко ждёт 5 клубник для пирога" | Обязательство |
| **Другой игрок приглашает** | Мультиплеер | "Alex пригласил на свой участок" | Социальное |
| **Событие/фестиваль** | Серверный календарь | "Фестиваль урожая в субботу!" | FOMO (мягкий) |
| **"Мир жил без тебя"** | Уведомления при входе | "Пока тебя не было: Лена заходила..." | Любопытство |

### 6.3 Что происходит при пропуске дня (дизайн "без наказания")

**Принцип:** Nookstead -- это тёплое одеяло, а не будильник.

| Что происходит | Результат | SDV для сравнения |
|---------------|---------|-------------------|
| Грядки не политы | Рост на паузе. Растения живы. | Растения увядают |
| Не зашёл 3 дня | NPC замечают: "Давно не видел, как дела?" | NPC не замечают |
| Не зашёл 7 дней | NPC немного "забывают" (memory decay), но не критично | Ничего |
| Не выполнил квест от доски | Квест пропадает, новый появляется. NPC не расстроен. | N/A |
| Не выполнил квест от NPC | NPC: "Ну ладно, может в другой раз..." Отношения не страдают. | Провал квеста |

**Тон NPC при возвращении:**
- 1 день: обычное приветствие
- 3 дня: "О, ты вернулся! Как дела? Всё хорошо?"
- 7 дней: "Куда ты пропадал? Я скучал! Тут столько всего произошло..."
- 14 дней: "Привет... Ты давно не заходил. Я немного забыл, о чём мы говорили... Расскажи!"

**Дизайнерская цель:** Каждый возврат ощущается как "прийти домой", а не как "вернуться к штрафам".

## 7. Мультиплеерные социальные механики

### 7.1 Оптимальный размер сервера: 5-10 активных игроков

GDD v3 предполагал 100 одновременных игроков. Revised roadmap сократил до 10-20 для MVP. С точки зрения game design, оптимум для cozy-игры -- **5-10 активных игроков на сервер**.

**Почему:**
- **Маленький город** -- в городке с населением 30-50 (NPC + игроки) все знают друг друга. Это создаёт ощущение сообщества, а не анонимной толпы.
- **NPC помнят всех** -- при 5-10 игроках каждый NPC имеет осмысленные воспоминания о каждом. При 100 игроках память разбавляется.
- **Аналогия: Animal Crossing** -- остров на 8 жителей + 1-3 друга = идеальный cozy-масштаб.

### 7.2 Как другие игроки усиливают cozy-опыт

| Механика | Описание | Эмоция |
|----------|---------|--------|
| **Видимость в городе** | Видишь, как другой игрок разговаривает с NPC или гуляет | "Я не один тут живу" |
| **NPC упоминает других** | Марко: "Вчера Alex приносил мне хлеб. У вас целый клуб пекарей!" | Социальная связь через NPC |
| **Посещение участков** | Зайти к другу, посмотреть ферму, оставить лайк | Взаимное восхищение |
| **Торговля предметами** | Обменяться урожаем напрямую | Кооперация |
| **Совместное подслушивание** | Два игрока рядом слушают NPC-NPC разговор | Shared experience |
| **Помощь с фермой** | Полить грядки друга (при визите) | Забота |
| **Совместные квесты** | "Принести 20 картофелин на городскую кухню" (от нескольких игроков) | Командная работа |

### 7.3 NPC как социальный клей между игроками

Ключевая инновация: NPC **связывают** игроков друг с другом, даже если те не общаются напрямую.

```
Игрок A подарил клубнику Марко
         │
         ▼
Марко запомнил: "Игрок A -- друг, приносит клубнику"
         │
         ▼
Марко в разговоре с Игроком B:
"Кстати, Игрок A вчера принёс мне клубнику. Хороший человек!"
         │
         ▼
Игрок B думает: "Кто такой Игрок A? Надо познакомиться!"
```

Это создаёт **опосредованную социальную связь** -- NPC сплетничают об игроках, создавая awareness и мотивацию для прямого общения.

### 7.4 Предотвращение токсичности

| Механизм | Описание | Эффективность |
|----------|---------|--------------|
| **Модерация чата** | Фильтр ключевых слов + rate limiting | Базовая защита |
| **NPC-реакция** | Если игрок груб с NPC, NPC "делится" с другими | Социальное давление через NPC |
| **Маленький сервер** | 5-10 игроков = каждый на виду | Трудно быть анонимно токсичным |
| **Нет PvP** | Нельзя украсть, сломать, навредить | Убраны все механики конфликта |
| **Позитивные механики** | Лайк участку, помощь с поливом, подарки | Reward за позитив, не за негатив |
| **Репорт-система** | Кнопка жалобы → модерация | Для крайних случаев |

**Философия:** В cozy-игре не нужны системы *наказания* за токсичность. Нужны системы *поощрения* за доброту. Если единственное, что ты можешь сделать другому игроку -- это помочь или похвалить, токсичность не возникает.

## 8. Системы прогрессии

### 8.1 Прогрессия фермы (горизонтальная + вертикальная)

**Горизонтальная:** больше грядок → больше культур → больше разнообразия.
**Вертикальная:** лучшие инструменты → меньше времени на рутину → больше времени на социал.

```
Неделя 1:  3x3 грядки, 3 культуры, базовые инструменты
           "Новичок-фермер"

Неделя 3:  4x4 грядки, 4 культуры, лейка Lv.2
           "Начинающий фермер"

Неделя 6:  5x5 грядки, 5 культур, все инструменты Lv.2
           "Опытный фермер"

Неделя 12: 6x6 грядки, все культуры, инструменты Lv.3
           "Мастер-фермер"
           Степан: "Лучший огород, что я видел за 40 лет!"
```

### 8.2 Прогрессия дома (Phase 2+)

| Уровень | Содержимое | Стоимость | NPC-реакция |
|---------|-----------|-----------|------------|
| Начальный | Пустой дом: кровать, стол | Бесплатно | Виктор: "Домик скромный, но уютный" |
| Обставленный | +5 предметов мебели | 200-1000 монет | Лена: "У тебя так красиво!" |
| Уютный | +10 предметов, декор стен | 1000-3000 монет | NPC хотят в гости |
| Роскошный | Полная обстановка, 2 этаж | 5000+ монет | "Лучший дом в городе!" |

### 8.3 Репутация в городе (упрощённая для MVP)

Вместо 30 уровней (GDD v3) -- **5 уровней**:

| Уровень | Название | Требование | Разблокировка |
|---------|---------|------------|--------------|
| 1 | Новичок | Начало игры | Базовый доступ |
| 2 | Житель | 7 дней в игре, 5 заданий | NPC здороваются первыми |
| 3 | Уважаемый | 21 день, 15 заданий, 2 NPC-друга | Доступ к бару (вечерние сцены) |
| 4 | Почётный | 45 дней, 30 заданий, 4 NPC-друга | NPC приходят с квестами, скидки |
| 5 | Легенда | 90 дней, все NPC -- друзья | Уникальные события, титул |

### 8.4 Коллекционирование

| Коллекция | Источник | Отслеживание | NPC-связь |

| Достижения | Milestone-события | Значки в профиле | NPC поздравляют |

## 9. Minimum Viable Cozy Loop

### 9.1 Формула

```
[Утренний ритуал]  →  [Продуктивная активность]  →  [Социальное взаимодействие]  →  [Milestone прогресса]  →  [Причина вернуться]
```

### 9.2 Заполненная формула для Nookstead

```
[Полить грядки,    →  [Продать урожай    →  [Поговорить с Марко,   →  [Клубника выросла! /  →  [Марко обещал
 собрать урожай,       на рынке,              подарить клубнику,        Купил расширение /       показать рецепт
 проверить почту]      купить семена,         услышать сплетню           Лена стала другом]      завтра. Тыква
                       выполнить задание]     Лены о Марко]                                      будет готова
                                                                                                  через 2 дня]
```

### 9.3 Абсолютный минимум механик для MVP

Если нужно выбрать ТОЛЬКО самое необходимое -- вот что создаёт "cozy magic":

| # | Механика | Зачем | Без неё |
|---|---------|-------|---------|
| 1 | **AI-диалоги с памятью** | Ядро USP. Без этого -- нет продукта. | Это просто Stardew клон |
| 2 | **NPC с расписанием и движением** | NPC должны ЖИТЬ на карте, не стоять столбами | Это Character.AI в картинках |
| 3 | **Фермерство (5 культур)** | Контекст для NPC-реакций + утренний ритуал + экономика | NPC не на что реагировать |
| 4 | **Рынок (продажа + NPC-комментарии)** | Экономическая цель + responsive moment | Нет причины фермить |
| 5 | **NPC-NPC разговоры** | "Город живой" + подслушивание (вирусный момент) | NPC -- чат-боты в вакууме |
| 6 | **Подарки NPC** | Прямое влияние на отношения + NPC запоминают | Нет способа развивать отношения кроме слов |
| 7 | **Уведомления "мир жил без тебя"** | Retention + отличие от Character.AI | Мир "мёртв" между сессиями |
| 8 | **Почтовый ящик** | Хук для возврата + async-коммуникация от NPC | Нет причины возвращаться |
| 9 | **Мультиплеер (видимость)** | "Я не один" + NPC упоминают других | Мир пустой |
| 10 | **День/ночь** | Ритм жизни, NPC меняют поведение, атмосфера | Статичный мир |

**Всё остальное** (декорации, профессии, крафт, транспорт, гильдии, сюжет, погода, животные) -- это расширение, не ядро.

### 9.4 "Первые 15 минут" -- определяют всё

Если Nookstead не зацепит за первые 15 минут, игрок не вернётся. Вот как должны пройти первые 15 минут:

```
Мин 0-1:   Ты прибываешь в Quiet Haven. Красивая анимация.
            Мэр Виктор встречает: "Добро пожаловать! Я покажу город."

Мин 1-3:   Мэр ведёт к пекарне. "Это Марко -- лучший пекарь."
            Марко: "О, новенький! *широко улыбается* Бери булочку! Бесплатно!"
            Игрок видит: NPC двигается, жестикулирует, у него есть личность.

Мин 3-5:   Мэр ведёт к парку. Лена поливает цветы.
            Лена: "*тихо, не отрывая взгляда от цветов* Привет... Ты тут новый?
            Красивый день для прогулки."
            Контраст с Марко: разные NPC -- разные характеры.

Мин 5-7:   Мэр приводит на участок. "Это твой дом! Тут есть грядки."
            Даёт 3 пакетика семян редиса и мотыгу.
            Игрок сажает первые 3 семечка (обучение фермерству).

Мин 7-10:  СВОБОДА. Мэр уходит. Игрок один на участке.
            Можно: вернуться в город, исследовать, поговорить с NPC.
            Большинство пойдут обратно к Марко или Лене -- "они были приятные".

Мин 10-13: ПЕРВЫЙ НАСТОЯЩИЙ ДИАЛОГ. Подходит к Марко.
            Марко помнит (!) -- "А, это ты! Как тебе булочка? Понравилась?"
            MAGIC MOMENT #1: NPC помнит то, что было 10 минут назад.
            Quick replies + свободный ввод. 3-5 реплик.

Мин 13-15: По дороге домой замечает: Марко и Ира разговаривают.
            Пузыри над ними: "Новенький, кажется, нормальный."
            MAGIC MOMENT #2: NPC обсуждают тебя за спиной.
            Эмоция: "Этот мир ЖИВОЙ."
```

**Результат:** За 15 минут игрок получил:
1. Ощущение живого мира (NPC двигаются, говорят, имеют характеры)
2. Первый момент "NPC запомнил!" (Марко помнит булочку)
3. Первый момент "NPC обсуждают меня!" (подслушивание)
4. Зерно фермерства (посажен редис, будет готов через 12 часов)
5. Причину вернуться (урожай + "что скажет Марко завтра?")

## 10. Анализ рисков

### 10.1 Матрица рисков по механикам

| Риск | Вероятность | Импакт | Описание | Митигация |
|------|------------|--------|---------|-----------|
| **Рост в реальном времени слишком медленный** | Средняя | Высокий | Игрок посадил картошку и ждёт 2 реальных дня. Скучно. Уходит. | Редис за 12 часов -- первый урожай быстро. Ежедневные задания + NPC-диалоги заполняют ожидание. Уведомления "Урожай готов!" привлекают обратно. |
| **AI NPC ломает иммерсию** | Средняя | Критический | NPC упоминает "игру", "ИИ", галлюцинирует, несёт бред. Один инцидент может стать вирусным (негативно). | Строгий system prompt. Пост-фильтрация. Кнопка репорта. A/B тестирование промптов. Мониторинг 100% диалогов первый месяц. |
| **Фермерство слишком просто** | Средняя | Средний | 5 культур, без крафта, без переработки. "Геймеры" разочарованы: "Это всё?!" | Для MVP -- ОК. Клубника + тыква + NPC-реакции добавляют глубину. Phase 2 добавляет 5+ культур. Фокус на social, не на farming. |
| **Фермерство слишком сложно** | Низкая | Средний | Casual-игроки (Мия, 31 год) не хотят разбираться в фермерстве. | 2x/день полив максимум. Нет удобрений в MVP. Нет болезней растений. Нет наказания за пропуск. Пропущенный полив = пауза, не смерть. |
| **Мультиплеер пустой** | Высокая | Высокий | Новый сервер с 1-2 игроками и 5-8 NPC. Город кажется пустым. | NPC *и есть* население. 5-8 NPC создают иллюзию жизни даже с 1 игроком. NPC-NPC разговоры, расписания, движение -- город живой. Рассчитывать на 3-5 активных игроков. |
| **"Я всё уже слышал" (AI-усталость)** | Средняя (после 2-4 нед.) | Высокий | Игрок замечает паттерны: Марко всегда предлагает хлеб, Лена всегда про цветы. | Инъекция событий (новые факты в память NPC). Сезонная смена контекста. Эмерджентные события. Новые NPC каждые 1-2 месяца. Развитие NPC через рефлексии (характер эволюционирует). |
| **Экономика не балансируется** | Низкая | Средний | Игрок слишком быстро или слишком медленно копит монеты. | Фиксированные цены в MVP (легко тюнить). Телеметрия: отслеживать баланс монет по когортам. Тюнинг за 1-2 дня (изменение конфига, не кода). |
| **Responsive moments слишком редкие** | Средняя | Высокий | Игрок не замечает, что мир "реагирует". Играет как в обычную ферму. | Гарантировать 2-3 responsive moments за сессию (по дизайну). Визуальные подсказки (иконка, звук) когда NPC "заметил" действие. Уведомления: "Марко заметил, что ты продал клубнику." |
| **Responsive moments слишком навязчивые** | Низкая | Средний | Каждый NPC комментирует каждое действие. Ощущение слежки. | Лимит: NPC упоминает действие 1 раз. Не все NPC видят всё (зоны осведомлённости). Вероятностная модель: 30-50%, не 100%. |
| **Gossip-система создаёт дезинформацию** | Низкая | Низкий | NPC передают искажённую информацию. Игрок путается. | Лёгкое искажение = фича (как в реальной жизни). Грубые ошибки фильтруются через шаблоны. Игрок может "исправить" NPC: "Нет, это не так!" |

### 10.2 Специфические риски по конкурентным аналогам

| Сценарий | Риск | Урок от конкурента | Решение Nookstead |
|----------|------|-------------------|-------------------|
| Пустой мир (как Palia при запуске) | Нет игроков → город пустой → скучно | Palia: 6M аккаунтов, но retention 15% на D7 | NPC = основное население. 0 игроков = всё ещё живой город. Маленькие серверы (5-10 игроков). |
| Repetitive NPCs (как SDV после 40 часов) | NPC повторяют одно и то же → теряется магия | SDV: скриптовые диалоги кончаются | AI = бесконечный контент. Но нужны инъекции событий, чтобы контекст менялся. |
| Casual отток (как Animal Crossing через 3 месяца) | Casual-игроки заходят, забывают, уходят | AC:NH: 43M продаж, но 60% бросили за 3 месяца | NPC *скучают* по игроку и пишут письма. Мягкий FOMO через любопытство, не через наказание. |
| Слишком shallow gameplay (как Character.AI) | "Это просто чат-бот с картинками" | Character.AI: 20M MAU, но 0 gameplay depth | Фермерство + экономика + квесты + мультиплеер = gameplay, не просто чат. |
| AI говорит что-то ужасное (как ранний Tay от Microsoft) | PR-кризис, негативная вирусность | Microsoft Tay: стал расистом за 24 часа | Строгий system prompt + pre-filter + post-filter + мониторинг + кнопка репорта. NPC -- милые персонажи, не генерал-боты. |

## Приложение B: Словарь терминов

| Термин | Определение |
|--------|-----------|
| **Responsive Moment** | Момент, когда NPC реагирует на конкретное действие игрока. Ядро USP Nookstead. |
| **Magic Moment** | Момент, когда NPC вспоминает факт из прошлого разговора. Вирусный триггер. |
| **Gossip Propagation** | Система распространения информации между NPC через NPC-NPC разговоры. |
| **Memory Injection** | Добавление шаблонной записи в память NPC без LLM-вызова ($0 cost). |
| **Event Injection** | Добавление нового факта в память всех NPC для создания свежести контента. |
| **Cozy Principle** | Дизайн-правило: никогда не наказывать игрока. Пауза вместо потери. Тепло вместо тревоги. |
| **NPC Awareness Zone** | Область, в которой конкретный NPC "видит" действия игрока. Не все NPC видят всё. |
