# Механика 09: Репутация и прогрессия

## Цель

Связать все активности игрока в единую долгосрочную траекторию развития с понятными вехами и unlock-эффектами.

## Игровой цикл

1. Игрок совершает полезные действия (торговля, квесты, помощь NPC).
2. Получает очки репутации и/или профильный прогресс.
3. Достигает уровня и открывает новый контент.
4. Возвращается в цикл с новыми возможностями.

## Основные правила

- Городская репутация является основной горизонтальной прогрессией.
- Профильные направления (фермер, повар, исследователь) дают вертикальную специализацию.
- Unlock не должен ломать баланс ранних игроков.
- Негативное поведение может снижать локальную репутацию у отдельных NPC.

## Состояния и данные

- `TownReputation`: level, xp, milestonesReached.
- `ProfessionProgress`: trackId, level, xp.
- `UnlockState`: features, transports, zones, perks.
- `SocialReputation`: npcId -> relationshipTier.

## Параметры баланса

- Вехи уровней: 5/10/15/20/25/30.
- Кривая XP: умеренно растущая (без резких стенок до midgame).
- Catch-up буст для отстающих игроков: +10%-25% к XP в лимитированном окне.

## MVP-срез

- 1 основной трек городской репутации.
- 2 unlock-порога.
- Интеграция с квестами, рынком и NPC-отношениями.

## Телеметрия

- `reputation_xp_gained`.
- `reputation_level_up`.
- `unlock_obtained`.
- `progress_stall_detected` (долгий период без роста).

## Риски и анти-абьюз

- Гринд без смысла: награды должны менять игровой маршрут, а не только числа.
- Snowball-эффект: мягкие catch-up механики.
- Фарм на одном действии: diminishing returns на повторяющийся паттерн.
