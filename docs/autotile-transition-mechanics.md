# Вычисление переходных тайлсетов (Material Transition Resolver)

## Контекст
Механизм автотайлинга при рисовании кистью в редакторе (или при генерации) работает в режиме "Single-Layer Material Resolver", где переходы между материалами формируются автоматически на основе соседних тайлов.

С удалением старой логики на базе `renderPriority` (функция `computeNeighborMaskByPriority`) система полностью опирается на пайплайн, задокументированный в `docs/design/design-013-map-lib-extraction.md` (в частности, в разделе `Component 8: core/material-resolver.ts` и `recomputeAutotileLayers`).

**Файл реализации:** `packages/map-lib/src/core/autotile-layers.ts`

## Бизнес-требования к логике (Алгоритм "Доминирующего соседа")

Когда материал клетки изменяется, для нее и ее 8 соседей пересчитываются:
1. **Маска автотайла**
2. **Используемый тайлсет (`tilesetKey`)**

Алгоритм определения тайлсета для любой затронутой клетки:

### 1. Оптимизация skip для same-material соседей
Перед пересчётом проверяется: если ячейка (a) имеет тот же материал, что рисуется, (b) не была напрямую закрашена, и (c) уже имеет `SOLID_FRAME` (маска 255 = все 8 соседей совпадают), — её можно пропустить, т.к. добавление ещё одного same-material соседа не изменит маску 255.

Если же ячейка НЕ SOLID (transition frame), она пересчитывается, т.к. новый same-material сосед может изменить её маску.

### 2. Вычисление маски (Direct Material String Comparison)
Маска вычисляется строго на основе совпадения материалов: бит соседа устанавливается в `1`, если `neighbor.terrain === current.terrain`. На приоритеты не смотрим. Все ячейки (включая напрямую закрашенные кистью) вычисляют маску нормально.

### 3. Определение "Доминирующего соседа"
Если клетка полностью окружена "своим" материалом (маска `255`), она получает свой базовый чистый тайлсет (`materialInfo.baseTilesetKey`).
Если маска `< 255` (то есть мы на границе с другим материалом), система находит **доминирующего чужого соседа** среди 8:
- Считается количество одинаковых "чужих" материалов вокруг.
- Выбирается материал с максимальным счетчиком. Если ничья — берётся первый попавшийся.

### 4. Поиск тайлсета в Transition Map (прямой + indirect)
Сформировав пару `(currentMaterial, dominantNeighborMaterial)`, система ищет подходящий тайлсет:

**Правила поиска перехода (в порядке приоритета):**

1. **Прямой переход:** `fromMaterialKey = current, toMaterialKey = dominantNeighbor` → используется найденный тайлсет.

2. **Indirect (промежуточный) переход:** Если прямой пары нет, ищем промежуточный материал B, такой что:
   - Существует тайлсет `current → B` (т.е. `fromMaterialKey = current, toMaterialKey = B`)
   - И существует тайлсет, связывающий B с dominantNeighbor (в любом направлении: `B → neighbor` или `neighbor → B`)

   В этом случае используется тайлсет `current → B`.

   **Пример:** deep_water рядом с grass, нет `deep_water:grass`. Есть `deep_water:water` и `water:grass`. Система находит B=water → deep_water использует тайлсет `deep_water:water`. На стороне grass аналогично найдётся `water:grass`. Шов визуально показывает water с обеих сторон.

3. **Fallback:** Если ни прямого, ни indirect перехода нет — используется базовый тайлсет клетки (`baseTilesetKey`).

## Критерии приемки (AC)
- [x] Функция `computeNeighborMaskByMaterial` вычисляет битовую маску, 100% опираясь на строгое совпадение строк `.terrain`.
- [x] Логика в `recomputeAutotileLayers` использует функцию `findDominantNeighbor`, чтобы получить целевой материал по краям границы.
- [x] Система делает lookup в словаре тайлсетов по ключу `currentMaterial:dominantNeighbor`.
- [x] Поддерживается Indirect Resolution: если нет прямой пары `A:C`, ищется промежуточный B через `findIndirectTileset`.
- [x] Same-material соседи с `SOLID_FRAME` пропускаются при пересчёте (оптимизация); non-SOLID same-material соседи пересчитываются.
- [x] ~~Solid Block Rule~~ **Отключено**: все ячейки (включая painted) вычисляют маску нормально через `computeNeighborMaskByMaterial`.
